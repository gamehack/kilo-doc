设置

VS Code 扩展可以通过设置窗口配置，点击 Kilo Code 中的齿轮图标打开。更改适用于所有扩展界面，包括侧边栏和 Agent Manager。

使用智能体配置

更改 Kilo 配置最快的方式是让智能体帮你做。智能体有一个内置技能，理解完整的 kilo.jsonc 模式，可以直接读取、创建和更新你的配置文件。

  • "将我的默认模型切换为 Claude Sonnet"
  • "禁用 OpenAI 和 Groq 提供商"
  • "设置一个 Figma 的 MCP 服务器"
  • "自动批准所有读取和 glob 操作"

配置文件位置

  • 全局配置：~/.config/kilo/kilo.jsonc — 适用于所有项目
  • 项目配置：项目根目录中的 kilo.jsonc，或 .kilo/kilo.jsonc
⚠️ 警告

如果将配置文件纳入版本控制，确保它们不包含 API 密钥或其他密钥。请使用环境变量来存储凭据。

推理块

推理块在 VS Code 聊天 UI 中默认展开。启用 自动折叠推理 或设置 auto_collapse_reasoning：

{
  "auto_collapse_reasoning": true
}

实验性功能

新扩展通过设置中的 实验性 标签暴露实验性功能：

  • 分享模式 — manual、auto 或 disabled
  • LSP 集成 — 向智能体暴露语言服务器诊断
  • 粘贴摘要 — 在包含大型剪贴板粘贴之前进行摘要
  • 批量工具 — 允许智能体在一个步骤中批量调用多个工具
  • OpenTelemetry — 启用 Kilo 遥测
